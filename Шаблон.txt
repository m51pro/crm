  ---
  Как это должно работать

  1) Документ строится из блоков

  Не просто свободный текст, а набор структурных блоков:
  - заголовок
  - абзац
  - таблица
  - переменная
  - блок на 2 колонки
  - блок подписей
  - реквизиты
  - разрыв страницы

  Пользователь не должен вручную “верстать” подписи. Он должен вставить готовый блок и заполнить его.

  ---
  2) Переменные вставляются как отдельные элементы

  Переменная должна быть не просто {{client_name}} в тексте, а визуальный inline-node.

  Поведение:

  - в редакторе выглядит как чип / тег
  - в preview подставляется реальным значением
  - если значения нет — подсвечивается как missing
  - можно вставлять через правую панель

  Примеры переменных:

  - client_name
  - client_passport
  - doc_number
  - doc_date
  - doc_amount
  - vat_amount
  - cottage_name
  - deal_start
  - deal_end
  - my_name
  - my_signatory_role

  ---
  3) Блок подписей делается отдельным node

  Это ключевое.

  Лучше всего:

  signatureBlock = таблица 2x1

  Внутри:
  - левая колонка:
    - “Исполнитель”
    - ФИО
    - подпись
    - печать
  - правая колонка:
    - “Заказчик”
    - ФИО
    - подпись
    - дата

  Почему таблица:

  - стабильно печатается
  - не ломается при переносах
  - легко держит 2 колонки
  - хорошо рендерится в HTML/PDF

  ---
  4) Блок “2 колонки” нужен отдельно

  Это второй важный node.

  Использование:

  - под подписи
  - для реквизитов
  - для двух блоков условий
  - для сравнения или разделения информации

  Поведение:

  - строго 50/50 или 60/40
  - внутрь можно вставлять обычный контент
  - при печати не должен распадаться

  ---
  Рекомендуемая структура конструктора

  Верхняя панель

  - Название шаблона
  - Сохранить
  - Предпросмотр
  - Экспорт PDF
  - Импорт DOCX
  - Настройки страницы

  Левая зона

  - сам редактор документа
  - лист A4
  - toolbar

  Правая зона

  - поиск по переменным
  - категории переменных
  - готовые блоки:
    - абзац
    - заголовок
    - таблица
    - 2 колонки
    - подписи
    - реквизиты
    - разрыв страницы

  ---
  Настройки, которые я бы заложил

  Параметры страницы

  - формат: A4
  - поля:
    - сверху: 20mm
    - снизу: 20mm
    - слева: 20mm
    - справа: 15mm
  - фон страницы: белый
  - бумага не должна быть “тяжёлой” визуально

  Typography

  - шрифт: Times New Roman для финального документа
  - шрифт интерфейса: Inter
  - размер текста документа: 14px
  - межстрочный интервал: 1.2

  Таблицы

  - border-collapse: collapse
  - border: 1px solid #000
  - padding: 8px
  - width: 100%

  Переменные

  - отображать как pill
  - цвет:
    - primary variables: синий
    - finance variables: зелёный
    - client variables: фиолетовый
    - signature variables: розовый/нейтральный

  Блок подписей

  - внешняя рамка: тонкая или без рамки
  - две равные колонки
  - под каждой подписью:
    - роль
    - линия
    - расшифровка
  - печать внизу или справа внутри блока

  ---
  Кейсы использования

  Кейс 1. Договор аренды

  Пользователь:
  1. вставляет заголовок
  2. добавляет блок “стороны”
  3. вставляет переменные клиента
  4. добавляет блок условий
  5. вставляет переменные дат и суммы
  6. внизу вставляет signatureBlock

  Результат:

  красивый договор с нормальной нижней частью и подписями в 2 колонки.

  ---
  Кейс 2. Акт выполненных работ

  Пользователь:
  1. вставляет заголовок акта
  2. вставляет таблицу услуг
  3. добавляет сумму, НДС, итого
  4. вставляет signatureBlock
  5. при необходимости добавляет requisitesBlock

  Результат:

  акт быстро собирается без ручной верстки.

  ---
  Кейс 3. Счёт

  Пользователь:
  1. вставляет реквизиты компании
  2. вставляет реквизиты клиента
  3. добавляет таблицу позиций
  4. вставляет итоговую сумму
  5. добавляет печать и подпись

  Результат:

  счёт получается структурным и аккуратным.

  ---
  Кейс 4. Шаблон с разными вариантами подписей

  Например:
  - арендодатель / арендатор
  - исполнитель / заказчик
  - компания / клиент

  Решение:

  signatureBlock должен принимать props:
  - leftTitle
  - rightTitle
  - leftName
  - rightName
  - showStamp
  - showDate

  Так один и тот же блок подходит для всех документов.

  ---
  Что не делать

  - не строить подписи через простые div/flex без структуры
  - не заставлять пользователя вручную собирать 2 колонки
  - не хранить всё как сырой HTML без блоков
  - не делать отдельный редактор для каждого типа документа
  - не использовать слишком много разных способов вставки одного и того же

  ---
  Итоговое решение

  Если кратко, лучший вариант такой:

  - Tiptap как база
  - custom nodes для структуры документа
  - таблица для подписей и колонок
  - Handlebars для подстановки данных
  - Playwright для PDF
  - shadcn/ui для удобного интерфейса

  Это даст:
  - удобство
  - контроль структуры
  - нормальные подписи
  - поддержку договора / акта / счета
  - одинаковый preview и финальный документ
